# Changelog

## 2.8.0
- ⚠️ **[중요 보안 정책] 외부망 접속 시 인증 토큰(?token=...) 필수화**: 사설망(가정 내부망)이 아닌 외부망(LTE/외부 와이파이 등)에서 웹 플레이어 화면(`3005` 포트)에 접속하려면 구성 탭의 토큰값 주입이 반드시 필요하며, 누락 시 진입이 차단(`403 Forbidden`)되고 5회 실패 시 5분 간 IP가 강제 차단(Ban)되도록 보안 대폭 강화.
- **무차별 웹 주소 스캔 공격(Brute-force) 차단**: 오인증 시 1초 인위적 지연 적용 및 Fail2Ban(5회 비인가 진입 실패 시 5분 간 임시 IP 영구 차단) 메커니즘을 전역 통합 가드로 구현.
- **PWA (Progressive Web App) 앱 연동 지원**: `manifest.json` 및 `service-worker.js` 설정을 추가하고 static 라우팅 기능을 활성화하여 모바일 기기 바탕화면 등에 독립형(Standalone) 앱처럼 설치 및 구동 가능하도록 지원.
- **스트리밍 통신 아키텍처 개량 (GET/POST 이원화)**: HTTP GET 스트리밍 요청에 대해 데이터 파싱 완료 이벤트를 기다리지 않고 무지연으로 즉시 스트림을 바이패스(Bypass)하도록 전면 개편하여, 초기 버퍼링 지연으로 인한 스피커 기기 재생 중단 및 튕김 현상 원천적 해소.
- **네트워크 보안 및 사설 IP 표준화**: 사설망 식별 방식을 RFC 1918 기준으로 정밀 표준화하여 도커 내부 가상 브리지 대역(`172.16.x.x` ~ `172.31.x.x`) 완벽 호환 및 통신 권한 유효성 강화.
- **오류 감지 및 디버깅 강화**: 방송국 실시간 스트림 주소 파싱 실패 시 `502 Bad Gateway` 상태 코드를 반환하도록 개선하고, FFmpeg 인스턴스의 표준 에러 스트림(stderr)과 비정상 종료(exit) 코드를 콘솔 로그에 즉각 표출하도록 개선.

## 2.7.9
- **컨테이너 실행 환경 안정화**: 윈도우 환경에서 빌드 시 `run.sh` 스크립트의 개행 문자(CRLF) 꼬임으로 인한 `FATAL tini` 실행 불능 오류를 방지하기 위해 Dockerfile에 `sed` 개행 세정 처리 로직을 보완하고 `.gitattributes` 개행 방식을 고정함.
- **사설 IP 대역(RFC 1918) 판별 고도화**: `isPrivateIP` 헬퍼 함수를 신설하여 도커 브리지 네트워크 대역(`172.16.x.x`~`172.31.x.x`)을 완벽하게 지원하고 외부 제어 권한을 사전에 올바르게 제한.
- **HTTP 라우팅 구조 전면 리팩토링**: 단일화된 보안 인증 가드 및 `protectedPaths`를 구현하고, 바디 파싱 완료 후 일괄 스위치 라우팅을 타도록 구조 정비.
- **보안 위협 조치 강화**: X-Forwarded-For 프록시 스푸핑 공격을 막기 위해 직전 소켓 IP가 사설 대역일 때만 프록시 헤더를 신뢰하도록 `isLocalRequest` 로직 보완 및 로그 출력 시 전역 토큰 노출 마스킹 처리 추가.
- **저장소 파일 구조 GitHub 동기화**: 중복 코드 및 임시 파일 정리, 프로젝트 전체 구조를 시각화한 `tree.md` 추가.
